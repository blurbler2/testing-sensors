# Software Architecture

- Project layout (relevant):
  - `Core/Inc`, `Core/Src` : application code, drivers, HAL glue.
  - `Drivers/` : CMSIS and vendor HAL libraries.
  - `CMakeLists.txt` and `cmake/` : build configuration for `arm-none-eabi`.

- Key modules:
  - `main.c` : system init (clock, peripherals), COM logging, main loop.
  - `bme280.c` / `bme280.h` : modular BME280 driver implementing datasheet
    calibration reads and integer compensation algorithms.
  - `stm32wbxx_hal_*.c` : HAL and startup code generated by CubeMX.

- I2C usage:
  - The driver exposes `bme280_init(I2C_HandleTypeDef*, addr)` and
    `bme280_read_compensated(...)` which perform register I/O via HAL
    `HAL_I2C_Master_Transmit` / `HAL_I2C_Master_Receive`.
  - HAL expects a left-shifted address: use `#define BME280_ADDR (0x76 << 1)`.

- Build & debug flow:
  1. Generate code in CubeMX (enable I2C1 PB8/PB9, set clock/clock config).
  2. Configure CMake kit to `arm-none-eabi-gcc` in VS Code.
  3. `CMake: Configure` then `CMake: Build` to compile.
  4. Flash with OpenOCD / ST-Link; use provided `Flash STM32WB55 with OpenOCD` task.
  5. Monitor serial at 115200 for sensor output.

- Testing & validation:
  - Verify I2C device ready with `HAL_I2C_IsDeviceReady` for `BME280_ADDR`.
  - After init, call `bme280_read_compensated` and assert temperature/pressure/humidity
    are within expected physical ranges.

- Documentation & style:
  - Public driver functions are documented with Doxygen-style docstrings.
  - Keep hardware-specific mappings in `README.md` and `docs/hardware-architecture.md`.
